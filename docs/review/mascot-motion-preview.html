Mascot Motion Preview

This is the review format I would use for motion: a standalone HTML preview with real looping animation. For the shipping app asset, I would not use GIF. I would keep the mascot body static and animate only the 4x4 spark using frame data or a tiny sprite atlas.

Review Format

Standalone HTML with canvas rendering. Easy to iterate, easy to open locally, and keeps the pixel grid exact during review.

Shipping Format

Frame-based spark data in code, or a tiny sprite-sheet PNG. Avoid GIF for product use because scaling and transparency are harder to control cleanly.

All cards show the working state only. Body is fixed 8x8, spark is animated 4x4.
