# Execution Plans

This directory is reserved for plan files that describe work the repository expects agents or humans to carry through over more than one round.

## Convention

- put active plans under `docs/exec-plans/active/`
- move finished plans to `docs/exec-plans/completed/`
- keep one plan per coherent slice
- link stable background material from `docs/references/` instead of copying large context into each plan

## Minimum Structure

Each plan should capture:

- the problem statement
- the intended end state
- the verification path
- open risks or blockers

Create the `active/` and `completed/` subdirectories when the first plan lands.
